## Technical SEO Checklist

This checklist applies to the fair-do.com Next.js deployment. Given the AGENTS.md instruction that this version has breaking changes from standard Next.js, verify API and file structure against the relevant guide in `node_modules/next/dist/docs/` before implementing any item programmatically.

1. **Title tags and meta descriptions on every page.** Every route must have a unique `<title>` (50–60 characters) and `<meta name="description">` (150–160 characters). Use the framework's metadata API (verify current API — `export const metadata` or `generateMetadata` as appropriate per installed version). Avoid duplicate titles across tutor profile pages and subject category pages.

2. **Open Graph and Twitter Card tags.** Every public-facing page — homepage, pricing, blog posts, tutor profiles — needs `og:title`, `og:description`, `og:image` (1200×630px), `og:url`, `og:type`, and `twitter:card`. These are mandatory for Mumsnet shares, which is a critical distribution channel for parent-facing content.

3. **Canonical tags.** Set `<link rel="canonical">` on every page. Tutor profile pages with URL parameters (e.g., `?source=referral`) must canonicalise to the clean URL. Prevent duplicate content penalties from pagination on subject search results.

4. **XML sitemap.** Generate a dynamic sitemap at `/sitemap.xml` covering: all static pages, all blog posts (with `<lastmod>` and `<priority>`), and all public tutor profile pages. Exclude admin routes, auth routes, and draft posts. Submit to Google Search Console and Bing Webmaster Tools on launch day.

5. **robots.txt.** Create `/robots.txt` that disallows crawling of `/admin`, `/api`, `/dashboard`, `/onboarding`, and any private session routes. Ensure `sitemap: https://fair-do.com/sitemap.xml` is declared at the bottom of the file.

6. **Core Web Vitals — Largest Contentful Paint (LCP).** The homepage and tutor search results are the critical paths. LCP target is under 2.5 seconds. Audit hero images for size and format (use AVIF or WebP), ensure above-the-fold content is not blocked by render-blocking scripts, and verify that font loading uses `font-display: swap`.

7. **Core Web Vitals — Cumulative Layout Shift (CLS).** Reserve explicit width and height on all images, including tutor profile photos. CLS above 0.1 harms rankings and is common on dynamically-loaded search result pages. Test with Chrome DevTools Lighthouse before launch.

8. **Core Web Vitals — Interaction to Next Paint (INP).** Audit the booking flow and tutor search for heavy client-side JavaScript. Long tasks above 50ms on the main thread degrade INP. Profile with Chrome Performance panel and defer non-critical JS (analytics, chat widgets).

9. **Structured data — EducationalOrganization schema.** Add JSON-LD `EducationalOrganization` schema to the homepage and about page. Include `name`, `url`, `logo`, `contactPoint`, `sameAs` (LinkedIn, Twitter/X, Facebook), and `description`. This increases eligibility for knowledge panel appearance as brand searches grow.

10. **Structured data — Course or EducationEvent schema on tutor profiles.** For each tutor's subject listing, consider `Course` schema with `provider`, `name`, `description`, and `offers` (with `price` and `priceCurrency`). This is a long-term investment — as crawl coverage of tutor profiles grows, rich results in search become possible.

11. **Structured data — FAQ schema on blog posts.** Every blog post that contains a list of common questions (Posts 5, 6, 9, 12 in this plan) should include FAQ schema to target "People Also Ask" boxes. This is the fastest route to additional SERP real estate for a new domain with low authority.

12. **HTTPS and security headers.** Verify that HTTPS is enforced on all routes (redirect HTTP to HTTPS). Add `Strict-Transport-Security`, `X-Content-Type-Options`, `X-Frame-Options`, and `Content-Security-Policy` headers. These are ranking signals and trust indicators; parents and tutors sharing payment and personal data will be deterred by browser security warnings.

13. **Hreflang (future-proofing).** fair-do is UK-only, but if any content targets `en-US` traffic from search, add `hreflang="en-GB"` to all UK-targeted pages to signal geographic intent. At minimum, set `<html lang="en-GB">` on every page.

14. **Breadcrumb schema and navigation structure.** Implement BreadcrumbList schema on blog posts and subject category pages. URL structure should follow: `fair-do.com/blog/[slug]` for posts, `fair-do.com/tutors/[subject]/[level]` for search. Flat, logical URL structure helps Googlebot crawl efficiently.

15. **Google Search Console setup and index monitoring.** Submit the sitemap, verify the property with HTML tag method (most reliable for Next.js deployments), monitor for crawl errors weekly in the first 90 days, and set up Core Web Vitals report alerts. Create a separate property for the `www` subdomain if used, to catch redirect chains.

## Backlink Strategy

Domain authority for fair-do.com will start from zero. The strategy prioritises earned editorial links (highest value), resource page placements (mid value), and community participation (lower value but volume). The goal for the first six months is 20–30 quality links, focused on DA 40+ sources.

### 1. TES (Times Educational Supplement) — DA 78

**Opportunity:** TES publishes regular features on private tutoring economics, tutor workload, and EdTech tools. The January 2025 VAT change story has a natural sequel in the platform commission story.
**Pitch:** Offer the data from Post 11 (UK tutoring market) as an exclusive data briefing before publication, with a byline from the fair-do founder. Frame as: "UK tutors are losing £800–£1,500/month to platform commission. We built the alternative." Target the features editor, not news. TES links drive meaningful referral traffic from the teacher and tutor audience.

### 2. Guardian Education — DA 95

**Opportunity:** The Guardian covered the VAT change extensively. The natural follow-up story is what happened to the families who left private school and are now seeking tutoring, and what the tutoring market looks like for them.
**Pitch:** Offer an op-ed from the fair-do founder: "The private tutoring market is broken in the same way Uber broke taxi economics — and it's time tutors got a fair deal." Guardian Education is sympathetic to platform labour economics stories. Aim for a comment piece rather than a news mention; comment pieces carry stronger editorial links.

### 3. Mumsnet — DA 82

**Opportunity:** Mumsnet's Talk boards have active threads on 11+, GCSE revision, and tutoring platforms. Brand mentions on Mumsnet threads are followed links from a DA 82 domain.
**Pitch:** This is not a traditional pitch. fair-do should have a Mumsnet Business account. When parents ask for tutoring platform recommendations on relevant threads, fair-do staff (or the founder) should respond honestly and helpfully, linking to relevant content (Post 6 on 11+, Post 7 on safeguarding). The post should disclose the affiliation. Mumsnet also runs a "Blogger Network" and "Product Testing" programme — a structured review opportunity worth pursuing in Month 3.

### 4. Which? — DA 88

**Opportunity:** Which? has published several pieces on tutoring platform fees and parent value. Post 3 (platform comparison) and Post 5 (tutor costs) are the assets to pitch.
**Pitch:** Approach the Which? Money and Family editors with the commission data: specifically the Tutorful service fee model and the annual cost differential for a family using a commission platform versus fair-do. Which? values independently verifiable data. If fair-do can provide primary data (e.g., a survey of tutors about platform commission) the story becomes significantly more compelling.

### 5. Schools Week — DA 56

**Opportunity:** Schools Week covers EdTech and the economics of education supply. The VAT story, the B2B school channel, and the platform commission model are all in scope.
**Pitch:** Offer a contributed piece on how schools are increasingly using tutoring platforms for catch-up programmes and what procurement officers should look for (no commission, transparent pricing, safeguarding standards). This targets the School tier directly and builds a relevant backlink.

### 6. Sutton Trust — DA 67

**Opportunity:** The Sutton Trust publishes research on private tutoring, social mobility, and education inequality. Their data is frequently cited in the national press and their "recommends" or "further resources" pages link to relevant platforms.
**Pitch:** Reach out to the research team with fair-do's access tier data — specifically the Free tier that allows low-income tutors to build a client base without upfront cost, and any data on tutor earnings distributions. Position fair-do as a case study in fairer platform economics. A mention in a Sutton Trust report would carry significant DA and editorial credibility.
